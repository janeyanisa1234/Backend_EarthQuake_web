earthquake.js 
    ทำหน้าที่สำคัญในการคำนวณความเสี่ยงแผ่นดินไหวสำหรับเขตต่าง ๆ ในกรุงเทพฯ โดยรวบรวมข้อมูลจากฐานข้อมูล ข้อมูลอาคารสูง รวมถึงข้อมูลแผ่นดินไหว (mock หรือจริง) 
    แล้วประมวลผลเพื่อประเมินระดับความเสี่ยง (Risk Level) ของแต่ละพื้นที่ ฟังก์ชันในไฟล์นี้ถูกออกแบบมาให้ทำงานร่วมกันอย่างเป็นระบบ ซึ่งสามารถแบ่งได้เป็นกลุ่มหลัก 
    เช่น กลุ่มสุ่มข้อมูล กลุ่มคำนวณทางภูมิศาสตร์ กลุ่มจัดการข้อมูลอาคาร กลุ่มดึงข้อมูลจากฐานข้อมูล และกลุ่มประมวลผลความเสี่ยง

⭐ 1. ฟังก์ชัน generateRandomId
    ฟังก์ชันนี้ทำหน้าที่สร้าง **รหัส ID แบบสุ่ม** ที่ประกอบด้วยตัวอักษร a–z, A–Z และตัวเลข 0–9 เพื่อใช้เป็น Primary Key 
    ของข้อมูลแผ่นดินไหวใหม่ทุกครั้งที่มีการบันทึกลงฐานข้อมูล จุดประสงค์ของการสุ่มคือเพื่อให้ ID ไม่ซ้ำกันแม้จะไม่มีการใช้ auto increment 
    จากฐานข้อมูลโดยตรง ฟังก์ชันจะวนสร้างตัวอักษรจำนวน 25 ตัว (หรือจำนวนที่กำหนด) แล้วคืนค่าเป็นสตริง ID หนึ่งตัว
⭐ 2. ฟังก์ชัน haversineDistance
    นี่คือฟังก์ชันสำคัญทางภูมิศาสตร์ที่ใช้คำนวณระยะทางระหว่างสองพิกัดละติจูดและลองจิจูด โดยใช้สูตร **Haversine Formula** 
    ซึ่งเป็นสูตรมาตรฐานในการวัดระยะทางบนพื้นผิวโลกที่มีความโค้ง มันคำนวณระยะเป็นหน่วยกิโลเมตร โดย: ฃ
    -> แปลงองศาเป็นเรเดียน
    -> คำนวณความแตกต่างของพิกัด
    -> คำนวณระยะโดยใช้รัศมีของโลก (6371 km)
    ผลลัพธ์ที่ได้จะถูกปัดให้เหลือทศนิยม 2 ตำแหน่ง
⭐ 3. ฟังก์ชัน countHighRiseBuildingsInDistrict
    ฟังก์ชันนี้มีหน้าที่ **นับจำนวนอาคารสูง (High-rise buildings)** ภายในระยะที่กำหนดจากจุดศูนย์กลางของแต่ละเขตในกรุงเทพฯ โดยถือว่าอาคารสูงคืออาคารที่มีจำนวนชั้น (building:levels) ตั้งแต่ 8 ชั้นขึ้นไป
    ฟังก์ชันจะทำงานโดย:
    -> วนตรวจสอบอาคารแต่ละหลังในข้อมูลที่โหลดมาจาก OpenStreetMap  
    -> ใช้ haversineDistance คำนวณระยะระหว่างอาคารกับเขต
    -> หากอาคารนั้นเป็น high-rise และอยู่ภายในรัศมีที่กำหนด (ค่าปกติคือ 10 กม.) จะถูกนับรวม
    ผลลัพธ์คือจำนวนอาคารสูงในเขตนั้น ซึ่งมีผลต่อการปรับระดับความเสี่ยงของแผ่นดินไหว เพราะเขตที่มีตึกสูงมากจะมีความเปราะบางต่อแรงสั่นสะเทือนมากขึ้น
⭐ 4. ฟังก์ชัน getBangkokDistricts
    ฟังก์ชันนี้ใช้สำหรับดึงข้อมูล **เขตในกรุงเทพมหานคร** จากฐานข้อมูล MySQL โดย lookup จากรหัสจังหวัด (province code = 10) ที่แทนกรุงเทพฯ
    มันจะเชื่อมต่อฐานข้อมูลแล้วดึงข้อมูลเขตทั้งหมด ได้แก่: id , name , latitude , longitude
    ข้อมูลเขตเหล่านี้เพื่อ:
    -> ใช้คำนวณระยะระหว่างแผ่นดินไหวกับเขต
    -> ใช้เป็นจุดอ้างอิงในการประเมินความเสี่ยง
    ถ้าหาไม่พบ ฟังก์ชันจะเตือนผ่าน console แต่ยังคงส่ง return ค่าเป็น array เปล่าเพื่อให้โค้ดส่วนอื่นไม่ล้ม
⭐ 5. ฟังก์ชัน loadBuildingsFromJson
    นี่คือฟังก์ชันที่เกี่ยวข้องกับการโหลดข้อมูลอาคารจากไฟล์ `export.json` ซึ่งเป็นไฟล์นำออกจาก OpenStreetMap (OSM) โดยฟังก์ชันนี้จะ:
    -> อ่านไฟล์ `export.json`
    -> แปลงข้อมูลเป็นโครงสร้าง JSON
    -> วนตรวจสอบ element ที่เป็นอาคาร (`type = "way"`)
    -> คำนวณจุด centroid ของอาคารแต่ละหลังด้วยฟังก์ชัน getCentroidCoordinates
    -> ดึงข้อมูล: height , building:levels
    -> ตรวจว่ามีจำนวนชั้นหรือไม่ ถ้าไม่มี → เก็บลงลิสต์ "อาคารที่ข้อมูลไม่ครบ"
    -> ตรวจว่าจำนวนชั้น ≥ 8 หรือไม่ เพื่อระบุว่าเป็น high-rise
    -> รวมข้อมูลทั้งหมดเข้าในอาร์เรย์ buildings
    ผลลัพธ์คือข้อมูลอาคารพร้อมคุณสมบัติที่จำเป็น เช่น: พิกัดเฉลี่ย (latitude, longitude), ความสูง, จำนวนชั้น, สถานะอาคารสูง (isHighRise) 
    ข้อมูลชุดนี้ใช้ประกอบการคำนวณความเสี่ยงของแต่ละเขตในกรุงเทพฯ
